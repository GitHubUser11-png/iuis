# Windows Forms UI Elements - Hardcoded Implementation

## Overview

The Integrated University Information System (IUIS) is built with **hardcoded Windows Forms elements**, including DataGridView controls, buttons, panels, and other UI components throughout the application. This document catalogs all the hardcoded UI elements across the three applications: Student App, Employee App, and Admin App.

---

## Core UI Factory

### AppDataGridViewFactory
**Location:** `src/IUIS.SharedUI/DataGridViews/AppDataGridViewFactory.cs`

A factory class providing consistent DataGridView styling and column creation methods used across the entire application.

#### Supported Column Types:
- **TextBoxColumn** - Plain text display
- **NumericColumn** - Formatted numbers (N2, N0, etc.)
- **CurrencyColumn** - Money values (C2 format)
- **DateColumn** - Date values (MM/dd/yyyy)
- **CheckBoxColumn** - Boolean values
- **ButtonColumn** - Action buttons
- **StatusColumn** - Status badges with color coding
- **ComboBoxColumn** - Dropdown selections

#### Styling Features:
- Consistent color scheme (Segoe UI font, dark header, alternating rows)
- Header styling (Dark gray background, white text, bold)
- Row height: 32px
- Alternating row colors for readability
- Border and grid styling
- Status-based cell coloring (Active, Inactive, Pending, Suspended)

#### Key Methods:
```csharp
CreateStyledDataGridView()           // Create new styled grid
AddTextBoxColumn()                   // Add text column
AddNumericColumn()                   // Add numeric column
AddCurrencyColumn()                  // Add currency column
AddDateColumn()                      // Add date column
AddCheckBoxColumn()                  // Add checkbox column
AddButtonColumn()                    // Add button column
AddStatusColumn()                    // Add status column
AddComboBoxColumn()                  // Add dropdown column
ApplyConsistentStyling()             // Apply styling to existing grid
GetStatusStyle()                     // Get color styling for status
ApplyStatusRowColoring()             // Color rows by status
CreateEmptyStatePanel()              // Create empty state message
```

## Color Scheme (Hardcoded RGB Values)

- **Primary Background:** `Color.FromArgb(249, 250, 251)` - Light gray
- **White Backgrounds:** `Color.White` or `Color.FromArgb(255, 255, 255)`
- **Headers:** `Color.FromArgb(31, 41, 55)` - Dark gray
- **Text:** `Color.FromArgb(17, 24, 39)` - Dark gray/black
- **Secondary Text:** `Color.FromArgb(107, 114, 128)` - Medium gray
- **Primary Button:** `Color.FromArgb(79, 70, 229)` - Indigo
- **Secondary Button:** `Color.FromArgb(229, 231, 235)` - Light gray
- **Grid Border:** `Color.FromArgb(209, 213, 219)` - Border gray
- **Alternating Rows:** `Color.FromArgb(249, 250, 251)` - Light gray

### Status Colors:
- **Active:** Green background `Color.FromArgb(220, 252, 231)` with dark green text
- **Inactive:** Gray background `Color.FromArgb(243, 244, 246)` with dark gray text
- **Pending:** Orange background `Color.FromArgb(255, 243, 224)` with dark orange text
- **Suspended:** Red background `Color.FromArgb(254, 226, 226)` with dark red text

## Implementation Notes

1. **All UI elements are hardcoded** - No designer files or XAML
2. **Consistent styling** applied via factory classes
3. **Modular approach** with reusable components
4. **Tab-based navigation** for complex modules
5. **Grid cell click handlers** for action buttons
6. **Color-coded status indicators** throughout
7. **Summary panels** for quick information display
8. **Filter and search controls** for data management
9. **Metric cards** for dashboard statistics
